# ESP32 MPU6050 BLE Device

ESP32 устройство с BLE подключением, передающее реальные данные с датчика MPU6050 (акселерометр + гироскоп).

## Подключение MPU6050 к ESP32

| MPU6050 | ESP32     | Описание                    |
|---------|-----------|-----------------------------|
| VCC     | 3.3V или 5V | Питание (проверьте версию датчика) |
| GND     | GND       | Земля                       |
| SCL     | GPIO 21   | I2C Clock (SCL)             |
| SDA     | GPIO 22   | I2C Data (SDA)              |
| INT     | GPIO 15   | Прерывание (опционально)    |

## Требуемые библиотеки

Установите следующие библиотеки в Arduino IDE:

1. **I2Cdev** - библиотека для работы с I2C устройствами
2. **MPU6050** - драйвер для MPU6050 (зависит от I2Cdev)

Эти библиотеки можно установить через Менеджер библиотек Arduino IDE (Sketch → Include Library → Manage Libraries).

Или скачайте напрямую:
- [I2Cdev](https://github.com/jrowberg/i2cdevlib)
- [MPU6050](https://github.com/jrowberg/i2cdevlib/tree/master/libraries/MPU6050)

## Данные, передаваемые по BLE

Устройство передает JSON с данными ориентации:

```json
{
  "yaw": 45.23,
  "pitch": -12.45,
  "roll": 5.67,
  "quat_w": 0.9876,
  "quat_x": -0.0123,
  "quat_y": 0.0456,
  "quat_z": -0.0789
}
```

- **yaw, pitch, roll** - углы Эйлера в градусах
- **quat_w, quat_x, quat_y, quat_z** - кватернион ориентации

## Калибровка

После первой загрузки вы можете увидеть неточные данные. Для калибровки:

1. Поместите датчик на ровную поверхность
2. Откройте Serial Monitor (115200 бод)
3. Запишите смещения гироскопа
4. Обновите значения в функции `mpu_initialize()`:

```cpp
mpu.setXGyroOffset(220);
mpu.setYGyroOffset(76);
mpu.setZGyroOffset(-85);
mpu.setZAccelOffset(1788);
```

## Примечания

- Если вы используете другой GPIO для прерывания MPU, измените `#define INTERRUPT_PIN 15` на нужный пин
- По умолчанию используются стандартные I2C пины ESP32 (GPIO 21/22)
- Для отладки откройте Serial Monitor на 115200 бод
